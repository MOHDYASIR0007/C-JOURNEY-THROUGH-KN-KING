CH 1

Introducing C

Concept
History
  • Bell Labs, early 1970s: UNIX rewrite need
  • Thompson → B language → Dennis Ritchie → C
  • K&R C: 1978 original book standard
  • C89/ANSI: first official ISO standard
  • C99: major evolution with new features
Concept
C Strengths
  • Efficiency: compiles to tight machine code
  • Portability: runs on any hardware
  • Low-level: direct memory/hardware access
  • Small core: compact, few keywords
  • Foundation: basis of C++, Java, Python
Pitfall
C Weaknesses
  • No type safety: compiler rarely enforces
  • No bounds checking: arrays silently overrun
  • Undefined behavior: legal code can explode
  • Pointer errors: hardest class of bugs
Concept
Using C Effectively
  • Adopt: consistent coding conventions
  • Avoid: overly clever tricks
  • Stick: to standard C89/C99 only
  • Use: standard library for common tasks
CH 2

C Fundamentals

Syntax
Minimal Program Form
  • directives → int main(void) → { statements }
  • #include <stdio.h>: include I/O header
  • int main(void): mandatory entry function
  • return 0: signals success to OS
Concept
Compilation Pipeline
  • Step 1 Preprocessing: handles # lines
  • Step 2 Compiling: source → object code
  • Step 3 Linking: object + libs → executable
  • gcc -o pun pun.c: compile and link together
  • cc pun.c: UNIX traditional compiler command
Syntax
Directives
  • #include <stdio.h>: include standard I/O
  • Directives: start with #, one line, no semicolon
  • Headers: declare library functions
Syntax
Variables & Constants
  • int height: declares integer variable
  • float profit: declares float variable
  • int i = 0: declaration with initialization
  • #define LIMIT 100: named constant macro
  • No #define semicolon: it is not a statement
Syntax
printf & scanf
  • printf("fmt", args): formatted output
  • scanf("fmt", &var): formatted input
  • &: address-of — required in scanf for scalars
  • %d: integer; %f: float in format strings
Syntax
Comments
  • /* ... */: C89 block comment
  • // ...: C99 single-line comment
  • Cannot nest /* */ comments
Pitfall
Critical Bugs
  • Missing & in scanf: writes to random address
  • Unterminated /* comment: swallows code
  • main ≠ MAIN or begin: program won't run
  • Mismatched format specifiers: garbage output
Concept
Identifiers
  • Letters, digits, underscore only
  • First character: must NOT be digit
  • Case-sensitive: Height ≠ height ≠ HEIGHT
  • C89 significant: first 31 chars; C99: 63 chars
Syntax
Program Layout Rules
  • Statements: end with semicolon
  • Compound stmt: uses { } not semicolon
  • Whitespace: ignored between tokens
  • One statement per line: recommended style
CH 3

Formatted I/O

Syntax
printf Conversion Specifiers
  • %d or %i: signed decimal int
  • %u: unsigned decimal int
  • %o: unsigned octal
  • %x / %X: hex lowercase/uppercase
  • %f: fixed decimal float (6 places default)
  • %e / %E: scientific notation
  • %g / %G: shorter of %e or %f
  • %c: single character
  • %s: string (until \0)
  • %%: literal % sign
Syntax
printf Flags & Width
  • %10d: right-justify in field of 10
  • %-10d: left-justify in field of 10
  • %010d: zero-pad to width 10
  • %+d: always show + or - sign
  • %.4f: 4 decimal places
  • %8.2f: width 8, precision 2
  • %*d: width from next argument
Syntax
scanf Conversion Specifiers
  • %d: reads int (decimal)
  • %f: reads float
  • %lf: reads double — l modifier required
  • %c: reads one char including whitespace
  • %s: reads word stops at whitespace
  • %[...]: reads matching char set (C89)
  • & required: for all scalar variables
Concept
Escape Sequences
  • \n: newline  \t: tab  \r: carriage return
  • \b: backspace  \a: alert/bell
  • \\: literal backslash  \": double quote
  • \': single quote  \0: null char (value 0)
Pitfall
printf/scanf Traps
  • Extra %d in printf: reads garbage off stack
  • Missing & in scanf: overwrites random memory
  • %f for double in scanf: WRONG — use %lf
  • scanf("%c"): reads leftover \n from buffer
  • Mismatch count vs format: undefined behavior
  • printf("%s", NULL): crash — null ptr
CH 4

Expressions

Syntax
Arithmetic Operators
  • +: unary plus or binary addition
  • -: unary minus or binary subtraction
  • *: multiplication
  • /: division — truncates for int operands
  • %: remainder — int operands only
Concept
Division Behavior
  • 7 / 2 = 3: fractional part silently dropped
  • C89: -9/7 can be -1 or -2 (impl-defined)
  • C99: -9/7 = -1 always (truncates toward zero)
  • 10 % 3 = 1: remainder sign = dividend sign in C99
  • i / 0: undefined behavior — not just an error
Concept
Operator Precedence
  • Highest: unary + -
  • Next: * / %
  • Lowest binary: + -
  • Left associative: all binary arithmetic ops
  • Use parentheses liberally when unsure
Syntax
Assignment Operators
  • i = 5: simple assignment (right-to-left)
  • i += 2 / -= / *= / /= / %=
  • Assignment is expression: v = e has value e
  • Chaining: i = j = k = 0: all set to 0
Syntax
Increment & Decrement
  • i++: post-increment — use i then add 1
  • ++i: pre-increment — add 1 then use i
  • i--: post-decrement  --i: pre-decrement
Concept
Lvalues & Expression Statements
  • Lvalue: expression designating object in memory
  • Variable i: is an lvalue; i + j: NOT an lvalue
  • Any expression + semicolon = statement
  • Side effects: assignment, inc/dec, function calls
Pitfall
Expression Traps
  • i / 0 or i % 0: undefined behavior
  • a[i] = i++: order of evaluation undefined
  • i++ * i++: two unsequenced side effects — UB
  • C89 impl-defined: -9/7 truncation direction
CH 5

Selection Statements

Syntax
Relational & Equality Operators
  • < > <= >= == !=
  • Returns: 0 (false) or 1 (true) — always int
Syntax
Logical Operators
  • &&: logical AND — short-circuit left to right
  • ||: logical OR — short-circuit left to right
  • !: logical NOT
  • Short-circuit: right side not eval if decided
  • !expr: 0 if expr nonzero; 1 if expr zero
Syntax
if Statement
  • if (expr) statement
  • if (expr) statement else statement
  • Cascaded if/else if/.../else chain
  • else: matches nearest unmatched if
Syntax
Conditional Expression
  • expr1 ? expr2 : expr3
  • Evaluates expr2 if expr1 nonzero, else expr3
  • i > j ? i : j: returns larger of i, j
  • Right-to-left associative
Syntax
switch Statement
  • switch (int-expr) { case c1: … default: … }
  • expr: must be integer type — not float or string
  • case values: must be integer constants
  • default: optional but recommended
  • break: required to stop fall-through
  • Fall-through: intentional only — comment it
Concept
C Boolean Values
  • Zero = false; any nonzero = true
  • C89: no boolean type — use int
  • C99: _Bool type — stores only 0 or 1
  • C99: #include <stdbool.h> gives bool/true/false
Pitfall
Selection Traps
  • if (i = 0): assignment not comparison!
  • Dangling else: else binds to nearest if
  • if without braces: only one statement controlled
  • i < j < k: means (i<j)<k NOT range check
  • Missing break: falls into next case silently
  • No ranges in cases: case 1..5: is illegal
CH 6

Loops

Syntax
while Loop
  • while (expr) statement
  • Tests condition BEFORE each iteration
  • If false initially: body never executes
  • while (1): infinite loop idiom
Syntax
do-while Loop
  • do statement while (expr);
  • Tests condition AFTER body executes
  • Body always executes at least once
  • Semicolon after ) is REQUIRED
Syntax
for Loop
  • for (init; test; update) statement
  • init: executes once before loop
  • test: checked before each iteration
  • update: executes after each iteration
  • for (;;): all parts optional — infinite loop
  • C99: int i = 0 allowed in init expr
Syntax
Comma Operator & Exit Statements
  • expr1, expr2: evaluates both; value = expr2
  • Lowest precedence; primary use: for loop init/update
  • for (i=0, j=n-1; i<j; i++, j--): example
  • break: exits innermost loop or switch only
  • continue: skips to next iteration of loop
  • goto label: avoid except for deeply nested breaks
Concept
Null Statement
  • ;: does nothing — valid statement
  • while (getchar() != '\n') ;: read to newline
  • Stray ;: if (i > 0); then next stmt always runs
Pitfall
Loop Traps
  • Off-by-one: fencepost error in bounds
  • Stray semicolon: for (...); empty body
  • Float loop counter: precision drift accumulates
  • Modifying loop variable inside body: confusing
  • Infinite loop: wrong condition or missing update
CH 7

Basic Types

Syntax
Integer Types C89
  • short int: ≥16 bits, signed
  • int: ≥16 bits (usually 32 on modern CPUs)
  • long int: ≥32 bits, signed
  • unsigned versions available for all
  • "int" keyword can be omitted in compound names
Syntax
Integer Types C99 Additions
  • long long int: ≥64 bits signed
  • unsigned long long int: ≥64 bits
  • _Bool: holds only 0 or 1
Concept
Integer Constants
  • 255: decimal  0377: octal (leading 0)  0xFF: hex
  • 255L: long  255U: unsigned  255LL: long long (C99)
  • 255ULL: unsigned long long (C99)
Syntax
Floating-Point Types
  • float: ~6-7 sig digits  3.14f: float literal
  • double: ~15-16 sig digits  3.14: double (default)
  • long double: extended precision  3.14L: literal
  • 1e10: scientific notation double
Syntax
char Type
  • char: stores one character — 1 byte
  • 'a': character constant in single quotes
  • char c = 'A': stores ASCII value 65
  • signed char vs unsigned char: impl-defined
  • unsigned char: use for raw byte manipulation
  • char arithmetic: chars are small integers
Concept
Type Conversions
  • Implicit: int + float → float result
  • Usual arithmetic: both operands promoted to wider type
  • (float) i: explicit cast to float
  • (int) 3.9: truncates to 3 — not rounds
  • char to int: sign extension if signed char
  • Narrowing: possible data loss — no warning by default
Syntax
typedef & sizeof
  • typedef int Bool: alias Bool for int
  • typedef unsigned char Byte: alias for byte
  • typedef: improves portability — not a new type
  • sizeof(int): bytes of type  sizeof(x): bytes of var
  • Returns: size_t — an unsigned type
  • sizeof array: total bytes of entire array
Pitfall
Type Traps
  • char signedness: varies — specify signed/unsigned
  • Signed integer overflow: undefined behavior
  • %f for double in scanf: WRONG — use %lf
  • Implicit narrowing cast: data loss silently
  • sizeof gives pointer size after array decay
CH 8

Arrays

Syntax
1D Array Declaration
  • int a[10]: array of 10 ints — indices 0 to 9
  • int a[5] = {1,2,3,4,5}: full initializer
  • int a[] = {1,2,3}: compiler infers size 3
  • int a[5] = {0}: all elements zero
  • int a[5] = {1}: only a[0]=1, rest 0
  • C99: int a[5] = {[2]=3}: designated init
Concept
Array Storage
  • Elements stored contiguously in memory
  • No runtime bounds checking — ever
  • Out of bounds: undefined behavior (silent)
  • sizeof(a)/sizeof(a[0]): portable element count
Syntax
Multidimensional Arrays
  • int m[3][4]: 3 rows × 4 cols = 12 elements
  • m[i][j]: row i, column j — zero-indexed
  • int m[2][3] = {{1,2,3},{4,5,6}}: 2D init
  • Stored: row-major order (row 0, row 1, ...)
Concept
VLA — Variable Length Arrays (C99)
  • int a[n]: n can be runtime variable
  • VLA: allocated on stack not heap
  • VLA: cannot have initializer
  • VLA: size fixed at point of creation
  • VLA: not allowed at file scope — only inside functions
Pitfall
Array Traps
  • a[10] for int a[10]: one past end — UB
  • No array assignment: a = b is illegal
  • Array name decays: a becomes &a[0] pointer
  • sizeof after decay: gives pointer size not array
  • Nested VLA: C99 only, no C89 support
Functions & Pointers — Ch 9–13
CH 9

Functions

Syntax
Function Definition
  • return-type name(params) { body }
  • int add(int a, int b) { return a+b; }
  • void f(void): no args, no return value
  • return expr: exits and returns value
  • return;: exits void function early
Syntax
Function Prototype / Declaration
  • int add(int, int);: prototype before use
  • Prototype: tells compiler argument and return types
  • C99 requires prototype before call
  • C89 allows implicit int: dangerous, avoid
  • Header files: collect prototypes for sharing
Concept
Argument Passing
  • C: call by value — all args are copies
  • Function cannot modify caller's original variable
  • To modify caller's var: pass pointer
  • Arrays: decay to pointer — modified in caller
Syntax
return Statement
  • return 0: main → success
  • return nonzero: main → failure signal
  • C99: falling off end of main = return 0
  • C89: falling off main = undefined return value
  • No return in non-void: returns garbage
Concept
Recursion
  • Function calls itself directly or indirectly
  • Must have: base case to terminate
  • Stack frame: added per recursive call
  • Deep recursion: stack overflow risk
  • Tail recursion: compiler may optimize
Pitfall
Function Traps
  • Missing prototype C89: args passed as int
  • Return local variable address: dangling ptr
  • No return in non-void: UB — compiler may warn
  • Recursion no base case: infinite stack overflow
  • K&R style params: outdated, avoid in new code
CH 10

Program Organization

Concept
Local Variables
  • Declared inside function body or block
  • Automatic storage duration: born on call, dies on return
  • Block scope: visible from declaration to end of block
  • C99: declare anywhere in block
  • C89: declare only at start of block
  • Parameters: same properties as local variables
Concept
Static Local Variables
  • static int count: persists across function calls
  • Initialized once: at program start
  • Block scope: still only visible in function
  • Use: maintaining state between calls
  • Not thread-safe: one copy shared by all calls
Concept
External (Global) Variables
  • Declared outside all functions
  • Static storage duration: live for program life
  • File scope: visible from declaration to file end
  • Accessible by all functions after its declaration
  • Initialized to zero automatically
Concept
Linkage
  • External linkage: visible across multiple files
  • Internal linkage: static global — file-private
  • No linkage: local variables
Concept
Canonical C Program Order
  • 1st: #include directives
  • 2nd: #define macros
  • 3rd: type definitions
  • 4th: external variable declarations
  • 5th: function prototypes except main
  • 6th: definition of main
  • 7th: definitions of other functions
Pitfall
Scope Traps
  • Global variable: any function can corrupt it
  • Local shadows global: same name hides outer
  • No extern in definition: linking confusion
CH 11

Pointers

Concept
Memory Model
  • Memory: sequence of bytes each with unique address
  • Variable: occupies one or more consecutive bytes
  • Address: location of first byte of variable
  • Pointer variable: a variable that stores an address
  • int *p: p can store address of any int
Syntax
Address & Indirection Operators
  • &i: address-of — gives address of variable i
  • *p: indirection — value at address in p
  • p = &i: p now points to i
  • *p = 5: writes 5 into i through p
  • int *p, *q: both p and q are pointers
Concept
Pointer Declaration Types
  • int *p  double *q  char *r
  • void *v: pointer to any type (generic)
  • NULL: null pointer constant (0 or (void*)0)
  • Always initialize pointer before use
Syntax
Pointers as Function Args
  • void inc(int *p) { (*p)++; }
  • inc(&i): passes address of i to function
  • (*p)++: parentheses required here
  • *p++: increments pointer p, not *p
Syntax
Returning Pointers
  • int *max(int *a, int *b)
  • NEVER return address of local variable
  • Return static or heap-allocated: safe
Pitfall
Pointer Traps
  • Uninitialized pointer: points to random memory
  • NULL dereference: *p when p==NULL → crash
  • Dangling pointer: points to freed/out-of-scope
  • *p++ vs (*p)++: entirely different operations
  • Aliasing: two ptrs modify same object unexpectedly
CH 12

Pointers and Arrays

Concept
Array Decay Rule
  • Array name → pointer to first element
  • int a[10]; int *p = a: same as p = &a[0]
  • WHY: C designed for fast array traversal
  • Exception: sizeof(a) gives full array bytes
  • Exception: &a gives pointer to whole array
  • Exception: string literal assignment
Syntax
Pointer Arithmetic — Three Forms Only
  • p + j: pointer to element j beyond p
  • p - j: pointer to element j before p
  • p - q: number of elements between p and q
  • p + 1: moves sizeof(*p) bytes forward
  • a[i] ≡ *(a+i): subscript = pointer arithmetic
  • i[a] ≡ *(i+a): also legal (rarely used)
Concept
Array Name as Pointer
  • *a = 7: same as a[0] = 7
  • *(a+1) = 12: same as a[1] = 12
  • for (p = a; p < a+N; p++): pointer loop
  • a = p: ILLEGAL — array name not assignable
Syntax
Pointers and 2D Arrays
  • int a[R][C]: stored row-major in memory
  • p = &a[0][0]: pointer to flattened array
  • a[i]: pointer to first element of row i
  • for (p=&a[0][0]; p<=&a[R-1][C-1]; p++)
Syntax
Pointers and VLAs (C99)
  • int (*p)[n]: pointer to VLA row of n ints
  • p = &a[0]: p points to first row
  • (*p)[i]: element i of current row
Pitfall
Pointer Arithmetic Traps
  • p - q unrelated pointers: undefined behavior
  • Pointer past one-beyond-end: legal to form, never dereference
  • int * vs int (*)[N]: very different types
  • Comparing pointers across arrays: UB
CH 13

Strings

Concept
String Representation
  • String: array of char terminated by '\0' (null)
  • '\0': null character, ASCII value 0
  • "hello": stored as h-e-l-l-o-'\0' = 6 bytes
  • String length: chars before '\0'
  • WHY null-terminated: no built-in string type in C
Syntax
String Literals & Variables
  • Adjacent literals: "ab" "cd" = "abcd" (concatenated)
  • String literal: stored in read-only memory
  • char s[10]: array to hold up to 9-char string
  • char s[] = "abc": compiler sizes it to 4
  • char *p = "abc": p points to read-only literal
  • p[0] = 'X': UB — literal is read-only
  • char s[] = "abc"; s[0]='X': OK — mutable copy
Syntax
String I/O
  • printf("%s", s): print until '\0'
  • scanf("%s", s): reads word — no & needed
  • gets(s): reads line — DANGEROUS no bounds
  • fgets(s, n, stdin): safe bounded line read
  • puts(s): print string then newline
  • fputs(s, stdout): print string no newline
Syntax
String Library <string.h>
  • strlen(s): count chars before '\0'
  • strcpy(dst, src): copy src into dst
  • strncpy(dst, src, n): safe — max n chars
  • strcat(dst, src): append src onto dst
  • strncat(dst, src, n): safe bounded append
  • strcmp(s1, s2): 0=equal <0=s1 less >0=s1 greater
  • strncmp(s1, s2, n): compare first n chars
  • strchr(s, c): ptr to first c in s or NULL
  • strstr(s1, s2): ptr to first s2 in s1 or NULL
  • sprintf(buf, fmt, ...): print to string buffer
Syntax
Command-Line Args
  • int main(int argc, char *argv[]): with args
  • argc: count of command-line arguments
  • argv[0]: program name string
  • argv[1]: first user argument
  • argv[argc]: always NULL pointer
Pitfall
String Traps
  • strcpy no size check: buffer overflow
  • gets: never use — removed in C11
  • strcmp returns 0 for equal: NOT true/false
  • strlen returns size_t: unsigned subtraction trap
  • char *p = "abc"; p[0]='X': crash on many systems
  • Forget \0: functions read past array end
Advanced Features — Ch 14–20
CH 14

The Preprocessor

Concept
How Preprocessor Works
  • Runs BEFORE compilation: text substitution
  • Handles: all lines beginning with #
  • Does NOT understand C syntax or types
  • Output: transformed C source fed to compiler
Syntax
Object-Like Macros
  • #define PI 3.14159: constant substitution
  • #define STR "hello": string constant
  • #undef PI: remove macro definition
  • No type checking: unlike const variables
  • Expands literally: watch precedence
Syntax
Function-Like Macros
  • #define MAX(a,b) ((a)>(b)?(a):(b))
  • MAX(x, y): expands inline — no function call
  • Parenthesize all params: prevents precedence bugs
  • Parenthesize whole body: same reason
  • WHY: SQ(a+b) → ((a+b)*(a+b)) not a+b*a+b
Syntax
Conditional Compilation
  • #if expr #ifdef NAME #ifndef NAME
  • #elif expr #else #endif: required to close
  • Header guard pattern: #ifndef _H #define _H ... #endif
Syntax
Predefined Macros
  • __LINE__: current source line number (int)
  • __FILE__: current source filename (string)
  • __DATE__: compilation date (string)
  • __TIME__: compilation time (string)
  • __STDC__: 1 if standard conforming compiler
  • __STDC_VERSION__: C99 = 199901L
Syntax
Other Directives
  • #error "message": force compile error
  • #pragma: implementation-specific instruction
  • #line n "file": reset line/file for diagnostics
  • Stringizing #: #define STR(x) #x → "x"
  • Token pasting ##: join two tokens into one
Pitfall
Macro Traps
  • No parens in body: operator precedence bugs
  • Side effects in args: SQ(i++) expands i++ twice
  • Multi-statement macro: wrap in do { } while(0)
  • Macro hides types: no type checking ever
  • Macro vs inline: C99 inline functions are safer
CH 15

Writing Large Programs

Concept
Source Files
  • .c files: contain function definitions and variables
  • One source file must contain main function
  • Each source file compiled separately
  • Then linked together into one executable
Concept
Header Files
  • .h files: shared information between source files
  • #include "myfile.h": include local header
  • Headers contain: prototypes, type defs, macros, extern
  • Headers do NOT contain: function definitions or storage
  • Header guards: prevent double-inclusion errors
Syntax
extern Keyword
  • extern int i: declares i defined elsewhere
  • extern: does NOT allocate storage
  • Definition: int i; — allocates storage once
  • Declaration: extern int i; — in all other files
  • Mismatch: extern long i; vs int i; → undefined
Concept
Module Strategy
  • Each module: one .c file + one .h file
  • .h file: interface — what callers need
  • .c file: implementation — how it works
  • Caller: #includes the .h file only
  • Design goal: minimize what's in .h files
Syntax
Building Multi-File Programs
  • gcc -o prog file1.c file2.c file3.c
  • Partial compile: gcc -c file1.c → file1.o
  • Link objects: gcc -o prog file1.o file2.o
  • Make utility: automates rebuild on change
  • gcc -DDEBUG=1: define macro on command line
Pitfall
Multi-File Traps
  • Define variable in header: multiple definition link error
  • No header guard: double-inclusion errors
  • Mismatch extern type vs definition: silent UB
  • Forget to recompile changed file: stale .o
CH 16

Structures, Unions & Enumerations

Syntax
Structure Basics
  • struct date { int d; int m; int y; }: tagged struct
  • struct date d1: declares variable of tagged type
  • d1.day = 15: dot operator accesses member
  • d1 = d2: struct assignment copies ALL members
Syntax
Structure Initialization
  • struct date d = {15, 6, 2025}: positional init
  • C99: struct date d = {.day=15, .month=6}: designated
  • Missing members: initialized to zero
Syntax
Structures & Functions
  • Pass struct by value: entire copy made
  • Pass struct by pointer: void f(struct date *p)
  • p->day: arrow — dereferences and accesses member
  • (*p).day: same as p->day
  • Return struct by value: copy returned
Syntax
Nested Structures & typedef
  • struct person { char name[40]; struct date dob; }
  • p.dob.year: chain dot operators
  • struct date dates[10]: arrays of structs
  • typedef struct { int x; int y; } Point;
  • Point p1: no need to write struct Point
Syntax
Union
  • union { int i; double d; float f; } u
  • All members: share same memory location
  • Only one member valid at a time
  • sizeof(union): size of largest member
  • Use: tagged union — store type along with value
Syntax
Enumeration
  • enum { MON=0, TUE, WED, THU, FRI, SAT, SUN }
  • enum color { RED, GREEN, BLUE }: named enum type
  • Values auto-increment from previous if unspecified
  • enum: more readable than bare #define integers
Pitfall
Struct/Union Traps
  • Padding: compiler adds bytes for alignment
  • sizeof(struct) ≠ sum of members: always use sizeof
  • Union: writing one member, reading another is UB in C89
  • Large structs by value: expensive copy — use pointer
  • Designated init: C99 only — not C89
CH 17

Advanced Uses of Pointers

Syntax
Dynamic Memory Allocation

Low-Level Programming

Syntax
Bitwise Operators — Six Total
  • ~i: bitwise complement — flip every bit
  • i & j: bitwise AND — 1 only if both bits 1
  • i | j: bitwise OR — 1 if either bit 1
  • i ^ j: bitwise XOR — 1 if bits differ
  • i << j: left shift i by j positions
  • i >> j: right shift i by j positions
Concept
Shift Semantics
  • i << j: equivalent to i × 2^j for unsigned
  • i >> j: equivalent to i ÷ 2^j for unsigned
  • Right shift signed: implementation-defined in C89
  • Portability rule: shift only unsigned operands
  • Shift by ≥ bit-width: undefined behavior
  • Negative shift amount: undefined behavior
Syntax
Bit Manipulation Idioms
  • SET bit k: i |= (1 << k)
  • CLEAR bit k: i &= ~(1 << k)
  • TEST bit k: (i >> k) & 1 or i & (1 << k)
  • TOGGLE bit k: i ^= (1 << k)
  • MASK low 4 bits: i & 0x0F
  • EXTRACT bits 4-7: (i >> 4) & 0x0F
Syntax
Bit-Fields in Structs
  • struct { unsigned int flag : 1; unsigned int val : 4; }
  • flag : 1: field occupies exactly 1 bit
  • Order: implementation-defined (not portable)
  • Cannot take address of bit-field with &
  • Use: compact storage for flags and small values
Concept
Low-Level Techniques
  • volatile: mark hardware registers, signal handlers
  • Union reinterpret: reading one member after writing another
  • Little-endian vs big-endian: byte order varies by CPU
  • typedef unsigned char Byte: portable byte type
Pitfall
Bitwise Traps
  • & vs &&: bitwise AND vs logical AND — very different
  • i << 32 for 32-bit int: undefined behavior
  • Right shift negative signed: impl-defined in C89
  • i & 0xFF == 0: parsed as i & (0xFF == 0) not (i & 0xFF) == 0
  • Bit-field layout: never assume portably

C89 vs C99 Quick Diff

C99
C99 New Features
  • // comments: C++-style line comments
  • Declare anywhere: not just block start
  • VLA: variable-length arrays
  • _Bool: boolean type
  • long long: 64-bit integer type
  • Designated initializers: .field = val
  • Compound literals: (int[]){1,2,3}
  • Flexible array members in structs
  • C99 division: always truncates toward zero
  • __func__: current function name predefined macro
  • inline functions: optimization hint
  • restrict qualifier: no-alias pointer hint
  • stdint.h / inttypes.h / stdbool.h added
  • complex.h / tgmath.h / fenv.h added
  • snprintf: safe sprintf with length limit
C89
C89 Restrictions to Know
  • Declarations: must come before first statement in block
  • No // comments: use /* ... */ only
  • No _Bool: use int with 0/1 convention
  • No VLA: all array sizes compile-time constant
  • Division -9/7: implementation-defined truncation
  • No designated initializers
  • No long long type
  • No inline keyword
  • implicit int return from functions: allowed but dangerous
